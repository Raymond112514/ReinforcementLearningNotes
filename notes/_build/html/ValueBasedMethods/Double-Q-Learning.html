8. Double Q Learning#

One challenge of deep Q-learning is that it often overestimates the action values. To understand why, consider a scenario where the action value for all states and actions is initially zero, i.e., \(Q(\mathbf{s}, \mathbf{a}) = 0\) for all \(\mathbf{s} \in \mathcal{S}\) and \(\mathbf{a} \in \mathcal{A}\). The update target should be:

\[y=r+\max_{\mathbf{a}\in \mathcal{A}} Q(\mathbf{s}', \mathbf{a}) = r\]

However, due to estimation errors, it is likely that for some state-action pairs \(Q(\mathbf{s}, \mathbf{a}) > 0\). Consequently, the update target becomes:

\[y=r+\max_{\mathbf{a}\in \mathcal{A}} Q(\mathbf{s}', \mathbf{a}) > r\]

As a result, the updated action value is overestimated. Over time, this overestimation bias accumulates and adversely affects the agent’s performance. To address this issue, Hasselt proposed Double Q-Learning, which uses two networks, \(Q_A\) and \(Q_B\), to estimate action values [1]. In particular, network \(A\) is used to find the action maximizer, while network \(B\) is used to evaluate the action value:

\[\mathbf{a}^*=\underset{\mathbf{a}}{\text{argmax}}\;Q_A(\mathbf{s}, \mathbf{a})\Longrightarrow Q_B(\mathbf{s}, \mathbf{a}^*)\]

The intuition is that even if \(Q_A\) has some estimation error, there is a chance that \(Q_B\)’s estimation error will counteract it. By disentangling the updates from biased estimates, Double Q-Learning helps to avoid maximization bias. The original algorithm proceeds as follows:

Algorithm 8.2 (Double Q Learning)

Inputs Separate action value \(Q_A, Q_B\)

  1. While episode \(n<N\):

    1. Select action \(\mathbf{a}\) based on \(Q_A\) and \(Q_B\)

    2. Choose randomly UPDATE_A or UPDATE_B

    3. If UPDATE_A

      1. Define \(\mathbf{a}^* = \underset{\mathbf{a}}{\text{argmax}}\;Q_A(\mathbf{s}, \mathbf{a})\)

      2. Update \(Q_A\)

      \[Q_A(\mathbf{s}, \mathbf{a}) \leftarrow Q_A(\mathbf{s}, \mathbf{a})+\alpha (r+\gamma Q_B(\mathbf{s}', \mathbf{a}^*)-Q_A(\mathbf{s}, \mathbf{a})\]

    4. If UPDATE_B

      1. Define \(\mathbf{a}^* = \underset{\mathbf{a}}{\text{argmax}}\;Q_B(\mathbf{s}, \mathbf{a})\)

      2. Update \(Q_A\)

      \[Q_B(\mathbf{s}, \mathbf{a}) \leftarrow Q_B(\mathbf{s}, \mathbf{a})+\alpha (r+\gamma Q_A(\mathbf{s}', \mathbf{a}^*)-Q_B(\mathbf{s}, \mathbf{a})\]

In the above algorithm, we need to train two action value function estimates, which can be computationally inefficient for deep neural network. Hassault, in another paper, proposed another double q learning scheme for deep neural network estimators [2]. The psuedocode is provided below

Algorithm 8.2 (Deep double Q Learning)

Inputs Primary network \(Q_\theta\), target network \(Q_{\theta'}\).

  1. While episode \(n<N\):

    1. Select action \(\mathbf{a}\) based on \(Q_A\) and \(Q_B\)

    2. Store transition \((\mathbf{s}, \mathbf{a}, r, \mathbf{s}')\) in replay buffer \(\mathcal{D}\).

    3. Sample a batch from \(\mathcal{D}\)

    4. Compute target Q-value

    \[ y = r + \gamma Q_{\theta}(\mathbf{s}, \underset{\mathbf{a}}{\text{argmax}}\;Q_{\theta'}(\mathbf{s}', \mathbf{a}))\]

    1. Perform gradient descent step on $\((Q_\theta(\mathbf{s}, \mathbf{a})-y)^2\)$

    2. Update target network parameters

    $$\theta’ \leftarrow \tau\cdot \theta + (1-\tau)\cdot \theta’

Note that in this updated scheme, we are training only one network. The weights of the target network are now a moving average of the primary network’s weights. This technique, known as Polyak averaging, improves stability during training. Below, we implement Double Q-Learning and test its performance on the CartPole and LunarLander environments, comparing it to Deep Q-Learning.

import gym
import pybullet_envs
import matplotlib.pyplot as plt
import pyvirtualdisplay
import imageio
import numpy as np
import torch
import torch.nn as nn
import torch.optim as optim
import torch.nn.utils as utils
from collections import deque
import copy
from torch.utils.data import Dataset, DataLoader
from utils import *

DEVICE = torch.device("cuda" if torch.cuda.is_available() else "cpu")

class Policy(nn.Module):
    """
    Implementation of policy 
    The policy contains a q_value function and a target q_value function, which is not being trained
    """
    def __init__(self, state_dim: int, action_dim: int, hidden_dim: int=128, device: str="cpu"):
        super(Policy, self).__init__()
        self.q_value = SimpleNeuralNetwork(in_dim=state_dim, hidden_dim=hidden_dim, out_dim=action_dim, n_layers=2)
        self.target_q_value = copy.deepcopy(self.q_value)
        self.action_space = torch.arange(action_dim)
        self.device = device

    def forward(self, state, target):
        state = torch.tensor(state).float().to(self.device)
        if len(state.shape) == 1:
            state = state.unsqueeze(0)
        if target:
            return self.target_q_value(state).detach()
        return self.q_value(state)

    def q(self, state, action, target):
        action = torch.tensor(action).to(DEVICE)
        q_values = self.forward(state, target)
        return q_values.gather(1, action.unsqueeze(-1).long())

    def select_action(self, state, epsilon=0.0):
        if torch.rand(1) < epsilon:
            return self.action_space[torch.randint(0, self.action_space.size(0), (1,)).item()].item()
        else:
            with torch.no_grad():
                return self.forward(state, target=False).argmax(1).item()

class DoubleQLearningAgent:
    """
    Implementation of double Q-learning agent
    """   
    def __init__(self, env, policy, buffer_max_len=10000, lr=1e-3, tau=1, device="cpu"):
        self.env = env
        self.policy = policy.to(device)
        self.buffer = ReplayBuffer(env, maxlen=buffer_max_len)
        self.device = device
        self.tau = tau
        self.optimizer = optim.Adam(self.policy.q_value.parameters(), lr=lr) 

    def learn_episode(self, batch_size=32, max_steps=10000, gamma=1, epsilon=0.1):
        self.policy.train()
        state = self.env.reset()
        done = False
        step = 0
        train_loss = 0.0
        train_reward = 0.0
        while not done and step < max_steps:
            # 1. Store transition into replay buffer, then sample a batch
            action = self.policy.select_action(state, epsilon=epsilon)
            next_state, reward, done, _ = self.env.step(action)
            train_reward += reward
            self.buffer.add(state, action, reward, next_state, done)
            states, actions, rewards, next_states, dones = self.buffer.sample(batch_size)
        
            # 2. Update primary network
            with torch.no_grad():
                opt_actions = self.policy(next_states, target=False).argmax(1)
                td_target = rewards + gamma * (1 - dones) * self.policy.q(next_states, opt_actions, target=True).squeeze()
            self.optimizer.zero_grad()
            loss = nn.MSELoss()(td_target.detach(), self.policy.q(states, actions, target=False).squeeze())
            loss.backward()
            self.optimizer.step()
            train_loss += loss.item()
            state = next_state
            step += 1
            
            # 3. Update target network via Polyak averaging
            for target_param, param in zip(self.policy.target_q_value.parameters(), self.policy.q_value.parameters()):
                target_param.data.copy_(self.tau * param.data + (1.0 - self.tau) * target_param.data)
        return train_loss, train_reward

We compared the performance of the Double Q-Learning agent with the Deep Q-Learning agent. Both agents successfully learned the CartPole task, and there does not appear to be a significant difference in performance between them.

env = gym.make('CartPoleBulletEnv-v1')
policy = Policy(env.observation_space.shape[0], env.action_space.n, device=DEVICE)
agent = DoubleQLearningAgent(env, policy, device=DEVICE)
history = agent.learn(n_episodes=500, batch_size=8, epsilon=0.5, print_every=50)

8.2. Lunar Lander#

We then trained both agents on the lunar lander environment, under the same hyperparameter setting as before. Both agents are trained for \(500\) epochs. It seems that double deep Q agent slightly outperforms deep Q agent after epoch \(200\).

env = gym.make("LunarLander-v2")
policy = Policy(env.observation_space.shape[0], env.action_space.n, device=DEVICE)
agent = DoubleQLearningAgent(env, policy, tau=0.0005, device=DEVICE)
history = agent.learn(n_episodes=500, batch_size=64, epsilon=0.3, print_every=10)
