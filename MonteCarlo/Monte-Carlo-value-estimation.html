5. Monte Carlo methods#

Not having access to the transition probabilities presents two challenges in finding the optimal policy:

  1. We can no longer apply dynamic programming methods like value iteration and policy iteration since both requires knowing the transition probability \(p(\mathbf{s}'|\mathbf{s},\mathbf{a})\)

  2. Knowing value functions \(V^*(\mathbf{s})\) is no longer sufficient for us to determine the optimal action, since the action value function also depends on \(p(\mathbf{s}'|\mathbf{s},\mathbf{a})\).

\[Q^*(\mathbf{s}, \mathbf{a}) = \sum_{\mathbf{s}'} p(\mathbf{s}'|\mathbf{s}, \mathbf{a}) [r+\gamma V^*(\mathbf{s})]\]

One way of coming around this problem is to directly estimate the \(V_{\pi}(\mathbf{s})\) and \(Q_{\pi}(\mathbf{s},\mathbf{a})\) using Monte Carlo methods. Since both functions are conditional expectation over rewards, we can estimate expectations through

\[\mathbb{E}[X] \approx \frac{1}{n}\sum_{i=1}^n X_i\]

Where \(X_i\) are samples from the distribution for \(X\). This suggests that we can estimate the value function by sampling from trajectories starting with \(\mathbf{s}_0=\mathbf{s}\) and then compute the culmulative reward.

5.1. First-visit v.s. every visit Monte Carlo#

There are two different variants of Monte Carlo methods:

  1. First-Visit Monte Carlo: In this approach, the value of a state is updated only the first time it is visited in each episode. Specifically, once a state is encountered, its value is updated based on the return from that first visit until the episode ends.

  2. Every-Visit Monte Carlo: Here, the value of a state is updated every time the state is visited within an episode. This approach averages the returns of all visits to the state throughout the episode.

We implement both variants in the next section.

5.2. BlackJack#

We test both algorithms in the Blackjack environment from OpenAI Gym. In Blackjack, the player’s goal is to achieve a hand value as close to 21 as possible without exceeding it. Each card in the deck has a specific value: number cards are worth their face value, face cards are worth 10, and aces can be valued as either 1 or 11. During gameplay, players decide between two actions: “hit” or “stick.” To “hit” means to draw an additional card to increase the hand’s value, while to “stick” means to keep the current hand and end their turn. Players compete against the dealer, who must draw cards until their hand totals at least 17.

5.3. MC Value Estimation Implementation#

We implement the value estimation algorithm for both first visit and every visit Monte Carlo method. We use the implementation to evaluate the strategy that stick at 20 or 21 and otherwise hit. The ran the algorithm for \(500000\) iterations. And the estimated value function is shown below.

class MonteCarloValue:
    """
    Monte Carlo method for estimating the value function
    """
    def __init__(self, states: list, policy: Policy):
        """ 
        @param states: A list of available states
        @param policy: A Policy instance with the sample method implemented
        """
        self.states = states
        self.policy = policy
        
    def estimate(self, env: any, n_episodes: int, gamma:float=1.0, every_visit:bool=False):
        """ 
        Estimates the value function
        @return value: The estimated value fucntion of the form {state1: value1,...}
        """
        value = {state: 0 for state in self.states}
        count = {state: 0 for state in self.states}
        for i in range(n_episodes):
            episode = self.policy.sample(env)
            states = list([state[0] for state in episode.keys()])
            cum_reward = 0
            for t, (state, reward) in enumerate(list(episode.items())[::-1]):
                state = state[0]
                t = len(episode) - t - 1
                cum_reward = gamma * cum_reward + reward
                if every_visit or state not in states[:t]:
                    count[state] += 1
                    value[state] = value[state] + (cum_reward - value[state]) / count[state]
        return value

policy = stick_at_20_or_21(states, actions)
mc = MonteCarloValue(states, policy)
values = mc.estimate(env, n_episodes=500000, every_visit=True)
plot_values(values)

One challenge with the Monte Carlo approach is that, when using a deterministic policy, it becomes difficult to estimate the action value function for unexplored states. This issue is particularly problematic in control problems where the goal is to derive the optimal policy, as unexplored states might have higher potential payoffs. We address this challenge in the next section.
