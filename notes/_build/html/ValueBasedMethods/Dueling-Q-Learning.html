9. Dueling Q Learning#

The dueling Q learning is proposed in 2015 by Wang et al. This method made a slight architectural changes to the traditional deep Q learning and shown great success. Recall that the advantage function is defined as

\[A(\mathbf{s}, \mathbf{a}) = Q(\mathbf{s}, \mathbf{a}) - V(\mathbf{s}) \implies Q(\mathbf{s}, \mathbf{a}) = A(\mathbf{s}, \mathbf{a}) + V(\mathbf{s})\]

The dueling architecture suggests separating the reprsentations of the value function and the advantage function. In particular, we model both of these function with a deep nueral network and add them together to get the action value fucntion to perform algorithms like Q learning.

Intuitively, this architecture helps since this helps the agent learn which states are useful without knowing the effect of taking each action. This is particularly useful for states where their actions do not affect the environment in a relevant way. In this case, it’s unnecessary to calculate the value of each action.

9.1. Identifiability#

One problem when the architure above is the lack of identifiability, that is, given a action value \(Q(\mathbf{s}, \mathbf{a})\), we cannot work it back to get \(A(\mathbf{s}, \mathbf{a})\) and \(V(\mathbf{s})\). TO solve this issue, the author considers the following modification over the action value

\[Q(\mathbf{s}, \mathbf{a}) = V(\mathbf{s}) + A(\mathbf{s}, \mathbf{a}) - \max_{\mathbf{a}'\in \mathcal{A}}A(\mathbf{s}, \mathbf{a}')\]

In this case, we can recover the value function by plugging in the action maximizer of the action function

\[Q(\mathbf{s}, \mathbf{a}^*) = V(\mathbf{s}) + A(\mathbf{s}, \mathbf{a}^*) - \max_{\mathbf{a}'\in \mathcal{A}}A(\mathbf{s}, \mathbf{a}')=V(\mathbf{s})\]

In the paper, the author proposed another objective. Instead of using the max-operator, it uses the mean operator. One benefit this introduced is more stabilized training empirically.

\[Q(\mathbf{s}, \mathbf{a}) = V(\mathbf{s}) + A(\mathbf{s}, \mathbf{a}) - \frac{1}{|\mathcal{A}|}\sum_{\mathbf{a}'\in \mathcal{A}}A(\mathbf{s}, \mathbf{a}')\]

Below we implemented the duel DQN architecture and test it performance on Cartpole enviroment.

import gym
import pybullet_envs
import matplotlib.pyplot as plt
import pyvirtualdisplay
import imageio
import numpy as np
import torch
import torch.nn as nn
import torch.optim as optim
import torch.nn.utils as utils
from torch.utils.data import Dataset, DataLoader
from utils import *

DEVICE = torch.device("cuda" if torch.cuda.is_available() else "cpu")

class Policy(nn.Module):
    def __init__(self, state_dim, action_dim, hidden_dim=128, device="cpu"):
        super(Policy, self).__init__()
        self.value = nn.Sequential(
            nn.Linear(state_dim, hidden_dim),
            nn.ReLU(),
            nn.Linear(hidden_dim, hidden_dim),
            nn.ReLU(),
            nn.Linear(hidden_dim, 1))
        self.advantage = nn.Sequential(
            nn.Linear(state_dim, hidden_dim),
            nn.ReLU(),
            nn.Linear(hidden_dim, hidden_dim),
            nn.ReLU(),
            nn.Linear(hidden_dim, action_dim))
        self.device = device
        self.action_space = torch.arange(action_dim)

    def forward(self, state):
        state = torch.tensor(state).float()
        if len(state.shape) == 1:
            state = state.unsqueeze(0)
        value = self.value(state)
        advantage = self.advantage(state)
        return value + (advantage - advantage.mean(1, keepdim=True))

    def q(self, state, action):
        action = torch.tensor(action)
        q_values = self.forward(state)
        return q_values[range(q_values.size(0)), action]

    def select_action(self, state, epsilon=0.0):
        if torch.rand(1) < epsilon:
            return self.action_space[torch.randint(0, self.action_space.size(0), (1,)).item()]
        else:
            with torch.no_grad():
                return self.forward(state).argmax(1)

class DuelQLearningAgent:
    def __init__(self, env, policy, lr=1e-3, device="cpu"):
        self.env = env
        self.policy = policy.to(device)
        self.buffer = ReplayBuffer(env, maxlen=10000)
        self.device = device
        self.optimizer = optim.Adam(self.policy.parameters(), lr=lr)

    def learn_episode(self, *param):
        batch_size, max_steps, gamma, epsilon = param
        self.policy.train()
        state = self.env.reset()
        done = False
        episode_loss, episode_reward, steps = 0.0, 0.0, 0.0

        while not done and step < max_steps:
            with torch.no_grad():
                action = self.policy.select_action(preprocess(state), epsilon=epsilon)
            next_state, reward, done, _ = self.env.step(action)
            episode_reward += reward
            self.buffer.add(preprocess(state), action, reward, preprocess(next_state), done)
            state_batch, action_batch, reward_batch, next_state_batch, done_batch = self.buffer.sample(batch_size)
            with torch.no_grad():
                target = reward_batch + gamma * (1 - done_batch) * torch.max(self.policy(next_state_batch), dim=1).values
            loss = nn.MSELoss()(target, self.policy.q(state_batch, action_batch))
            self.optimizer.zero_grad()
            loss.backward()
            self.optimizer.step()
            episode_loss += loss.item()
            state = next_state
            step += 1
        return episode_loss, episode_reward

env = gym.make('CartPoleBulletEnv-v1')
policy = Policy(env.observation_space.shape[0], env.action_space.n)
ql = DuelQLearningAgent(env, policy, lr=5e-4)
history = ql.train(n_episodes=1000, evaluate_n_episodes=1, epsilon=0.3, print_every=50)
